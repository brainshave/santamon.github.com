Stateful GUI FTW!

First post in English. Prepare for more.

From several commits back, GUI FTW! doesn't return single element after creation of GUI. Now you'll get a map wrapped in an atom. 

user> ((swing [JFrame [*id :window
                       *groups [:containers]]
               [JPanel [*groups [:containers :asdfs]]
                [JButton [*id :button
                          *groups [:asdfs]]]]]))

#Atom@533e846f:
  {:ids {:button #<JButton>
         :window #<JFrame>}
   :groups {:containers (#<JFrame> #<JPanel>)
            :asdfs (#<JPanel> #<JButton>)}
   :root #<JFrame ...>}

I call it "state map" because it holds information about runtime state. State map have three standard keys:

  • :ids — objects identified by their *id
  • :groups — objects grouped by their *groups
  • :root — root (top) element of gui structure — JFrame instance in this case.
